# Integrate KubeSlice-generated Events on Slack

This topic describes how to create an app in your Slack workspace and integrate the 
kube-state-metrics (KSM) events and KubeSlice events for the Slack channel.


## Create an App

Create an app using the Slack API in your workspace. 

To create an app:

1. Open the [Slack API](https://api.slack.com/apps) console.

2. Click the **Create an App** button.

   ![events](/images/version1.0.0/monitoring-events/kubernetes-events1.png)

3. Select **From an app manifest** to configure your app scopes and settings.
   
   ![events](/images/version1.0.0/monitoring-events/kubernetes-events2.png)

4. Select a workspace where you want to create an app and click **Next**.

   ![events](/images/version1.0.0/monitoring-events/kubernetes-events3.png)

5. In the **Enter app manifest below** section, click on the **YAML** tab and enter the following manifest:

   ```yaml
   display_information:
     name: Botkube
     description: Botkube
     background_color: "#a653a6"
   features:
     bot_user:
       display_name: Botkube
       always_online: false
   oauth_config:
     scopes:
       bot:
         - channels:read
         - app_mentions:read
         - chat:write
         - files:write
         - users:read # Remote configuration only: Used to get Real Name for audit reporting
   settings:
     event_subscriptions:
       bot_events:
         - app_mention
     interactivity:
       is_enabled: true
     org_deploy_enabled: false
     socket_mode_enabled: true
     token_rotation_enabled: false
   ```

6. Click **Next**.

7. In the **Review summary & create your app** section, review the details and click **Create**.

   ![events](/images/version1.0.0/monitoring-events/kubernetes-events4.png)


## Install an App

After you create an app, you will be redirected to an application details page where you can install an app in your workspace.

To install an app to your workspace:

1. Go to **Settings** > **Basic Information** on the left sidebar.

2. In the  **Building Apps for Slack** section, under **Install your app**, click the **Install to workspace** button.

3. Click **Allow** to finish the installation.


## Get an OAuth Token

While configuring botkube in the Slack, the OAuth token is necessary for authentication.

To get an oauth token for your workspace:

1. Go to **Features** > **OAuth & Permissions** on the left sidebar.

2. Copy the **Bot User OAuth Token** and save the token for later use while deploying the **kubeslice-botkube** 
  bundle in the controller and worker clusters.
   
    ![events](/images/version1.0.0/monitoring-events/bot-user-oauth-token.png)


## Get an App-Level Token

Slack App with Socket Mode requires an App-Level token for the websocket connection.

To get an app-level token:

1. Go to **Settings** > **Basic Information** on the left sidebar.

2. Click the **Generate Token and Scopes** button.
   
   ![events](/images/version1.0.0/monitoring-events/app-level-token.png)

3. In the **Generate an app-level token** section, enter a token name. For example, `botkube` is a token name.

   ![events](/images/version1.0.0/monitoring-events/generate-app-level-token.png)

4. Click **Add Scope** and select **connections:write scope**.

5. Click **Generate**.

6. Review the details and click **Done**. Copy the app-level token for later use while deploying the 
   **kubeslice-botkube** bundle in the controller and worker clusters.

## Create Slack Channel for Events
After you install a Botkube application, you see a new bot user with the name `botkube` added in your workspace. 
Now create the two channels to receive KSM and KubeSlice events in the workspace.

![events](/images/version1.0.0/monitoring-events/create-channel1.png)

Add the bot to a Slack channel you want to receive notification in. You can add it by inviting the **@Botkube** user to a channel.

![events](/images/version1.0.0/monitoring-events/add-bot-to-channel.png)

## Deploy the kubeslice-botkube Bundle

To deploy the **kubeslice-botkube** bundle:

1. Switch the context to controller cluster.
   
   ```
   kubectx <cluster_name>
   ```

2. Add the **kubeslice-botkube** repo using the following command:
   
   ```
   helm repo add kubeslice/botkube
   ```

3. Use the following command to create the `botkube` namespace on the controller and worker clusters.
   
   ```
   kubectl create ns botkube
   ```

4. Use the following command to deploy the **kubeslice-botkube** bundle on the controller cluster:
 
   :::note
   Replace the `channel_name`, `cluster_name`, `app_token`, and `bot_token` in the command below with your values.
   ::: 

   ```
   helm install kubeslice-botkube kubeslice/botkube  --set "sources.k8s-controller-events.botkube/kubernetes.enabled=true" --namespace botkube \
   --set communications.default-group.socketSlack.enabled=true \
   --set communications.default-group.socketSlack.channels.default.name=ksm-channel-name \
   --set communications.default-group.socketSlack.channels.kubeslice.name=kubeslice-events-channel-name \
   --set communications.default-group.socketSlack.appToken=App-token \
   --set communications.default-group.socketSlack.botToken=Bot-token \
   --set settings.clusterName=Cluster-name \
   --set 'executors.k8s-default-tools.botkube/kubectl.enabled'=true \
   --set 'executors.k8s-default-tools.botkube/helm.enabled'=true
   ```

5. Use the following command to deploy the **kubeslice-botkube** bundle on the worker cluster:
   
   :::note
   Replace the `channel_name`, `cluster_name`, `app_token`, and `bot_token` in the command below with your values.
   :::

   ```
   helm install kubeslice-botkube kubeslice/botkube  --set "sources.k8s-worker-events.botkube/kubernetes.enabled=true" --namespace botkube \
   --set communications.default-group.socketSlack.enabled=true \
   --set communications.default-group.socketSlack.channels.default.name=ksm-channel-name \
   --set communications.default-group.socketSlack.channels.kubeslice.name=kubeslice-events-channel-name \
   --set communications.default-group.socketSlack.appToken=App-token \
   --set communications.default-group.socketSlack.botToken=Bot-token \
   --set settings.clusterName=Cluster-name \
   --set 'executors.k8s-default-tools.botkube/kubectl.enabled'=true \
   --set 'executors.k8s-default-tools.botkube/helm.enabled'=true
   ```

After you deploy the charts in the workspace, you get the notifications.
